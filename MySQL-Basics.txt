Joins
      Inner Join
            Intersection. Matching rows between the two(JOINED) tables only are retrieved.             SELECT .......
            FROM tbl1             JOIN tbl2
            ON tbl1.dept_id = tbl2.id
            If the column name to be joined on is same in both tables,             SELECT .......
            FROM tbl1
            JOIN tbl2
            ON dept_id

      Outer Join
            Matching rows between the two(JOINED) tables + non matching rows from the left
SELECT .......             FROM tbl1
            LEFT JOIN tbl2
            ON tbl1.dept_id = tbl2.id
            // This will retrieve all rows from tbl1 + matching rows between tables tbl1 and tbl2.             // for non matching results, the respective column (if selected) will have NULL value.
            // similarly RIGHT JOIN.

            Full Outer Join.
SELECT .......             FROM tbl1
            FULL JOIN tbl2
            ON tbl1.dept_id = tbl2.id

      Cross Join
            Doesn't have ON clause.
            Gives cartesian product of the number of rows in both the tables.
            e.g. tbl1 has 10 rows, and tbl2 has 4 rows, then a cross join between them would result in 40 rows.
            SELECT .......
            FROM tbl1
            CROSS JOIN tbl2

       IMP - for UNION and UNION ALL to work, the number, datatypes, and the order of the
 columns in both SELECT stmt should be same.
       DIFFERENCES -
             UNION removes duplicate rows whereas, UNION ALL doesn't.
             UNION has to perform distinct sort to remove duplicates, which makes it less faster
 than UNION ALL.
       ORDER BY should be only at the last/end of SELECT stmts of UNION chain.
             SELECT * FROM tbl_india_customers
             UNION
             SELECT * FROM tbl_US_customers
             ORDER BY ....

UNION combines the rows from the tables, WHEREAS, JOIN retrieves the data
based on the logical relationship between the tables.
